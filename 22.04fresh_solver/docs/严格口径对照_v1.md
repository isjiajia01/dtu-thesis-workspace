# 严格口径对照 v1：Herlev / East / West

## 1. 目的
在当前 optimistic rolling baseline 的基础上，引入一版 `stricter realism config v1`，用更保守的 controller admission、更强的 depot-risk 惩罚、更小的 route packing 强度，以及更强的 repair penalty，观察三个实例在更严口径下的服务率与执行风险变化。

对应脚本：
- `src/algorithms/fresh_solver/julia/scripts/run_multiday_strict.jl`

---

## 2. stricter realism config v1 做了什么

### Controller 更保守
- `protected_reserve_ratio: 0.20 -> 0.28`
- `hard_protected_ratio: 0.40 -> 0.55`
- `flex_admission_cap_ratio: 0.90 -> 0.72`
- `risky_flex_cap_ratio: 0.25 -> 0.12`
- feedback thresholds 提前触发
- feedback clip step 更大

### Routing 更保守
- `max_orders_per_route: 8 -> 6`
- `depot_proxy_penalty: 2.0 -> 4.0`
- `trip2_penalty: 10.0 -> 18.0`
- `depot_service_buffer_min: 5.0 -> 12.0`
- split/refill 权重向 depot-safe 偏移

### Repair 更强调 depot 风险
- `gate_penalty_weight: 100 -> 180`
- `picking_colli_penalty_weight: 4 -> 10`
- `picking_volume_penalty_weight: 8 -> 16`
- `staging_penalty_weight: 6 -> 14`

## 3. 结果总表

| Instance | Baseline / Fast SR | Strict SR | Assigned Δ | Expired Δ | Deferred events Δ |
|---|---:|---:|---:|---:|---:|
| Herlev | 99.90% (`1043/1044`) | 98.75% (`1031/1044`) | -12 | +12 | `219 -> 915` |
| East | 100.00% (`3268/3268`) | 100.00% (`3268/3268`) | 0 | 0 | `2098 -> 3304` |
| West | 99.89% (`4546/4551`) | 99.63% (`4534/4551`) | -12 | +12 | `2829 -> 4639` |

## 4. 逐实例解释

## 4.1 Herlev
### 变化
- `service_rate: 0.9990 -> 0.9875`
- `unique_assigned_orders: 1043 -> 1031`
- `unique_expired_orders: 1 -> 13`
- `total_deferred_events: 219 -> 915`
- `max_overloads: 7 -> 3`

### 解释
Herlev 对 stricter realism config 有明显反应：
- 服务率下降约 1.15 个百分点；
- expired 从 1 增到 13；
- deferred 事件大幅增加；
- overload bucket 数下降。

这说明 Herlev 的 optimistic baseline 确实部分受益于较松的 admission / route packing / depot-risk 口径。

### 结论
Herlev 的高 SR 不是完全虚高，但在更严口径下会真实下降，因此更适合作为“高但非满分”的严格口径基线。

---

## 4.2 East
### 变化
- `service_rate: 1.0 -> 1.0`
- `unique_assigned_orders: 3268 -> 3268`
- `unique_expired_orders: 0 -> 0`
- `total_deferred_events: 2098 -> 3304`
- `max_overloads: 15 -> 14`

### 解释
East 在更严口径下仍然保持 100% unique service，但方式明显变了：
- controller 更积极 defer；
- 最终仍能在 horizon 内全部清空；
- 说明 East 更像“容量确实足够，但需要更多滚动缓冲”的 regime。

### 结论
East 的 100% SR 在当前模型下比 Herlev / West 更稳，不是轻微收严就会掉下去。但这仍不自动等于最终 business-feasible 100%，因为当前 depot realism 仍未 fully harden。

---

## 4.3 West
### 变化
- `service_rate: 0.9989 -> 0.9963`
- `unique_assigned_orders: 4546 -> 4534`
- `unique_expired_orders: 5 -> 17`
- `total_deferred_events: 2829 -> 4639`
- `max_overloads: 26 -> 24`

### 解释
West 对 stricter realism config 也有实质反应：
- 服务率下降；
- expired 增加；
- deferred 增加明显；
- overload bucket 略降。

这与我们此前对 West 的判断一致：West 是更敏感的 regime，严口径一上去就更容易牺牲服务率。

### 结论
West 的 optimistic baseline 明显比 East 更脆弱，因此论文里更适合把 West 作为“结构与 realism trade-off 最清楚”的主分析对象。

---

## 5. 这次对照说明了什么

## 5.1 optimistic baseline 确实偏乐观，但没有完全塌掉
如果 optimistic baseline 是纯假象，那么 stricter realism 一上去应该会：
- SR 大幅坍塌
- backlog 堆积严重
- final carryover 大量残留

当前并没有发生这种情况。说明：
- 主架构不是错的；
- 当前 solver 的高服务能力有相当一部分是真实力；
- 但 optimistic baseline 的确高估了最终可执行表现，尤其在 Herlev / West 上更明显。

## 5.2 严口径当前主要体现在“更多 deferred / 更多 expired”
而不是直接把所有实例都打崩。
这意味着更严口径主要通过：
- 更早 admission clipping
- 更少 risky flex
- 更保守 route packing

来实现，而非从根本上让 routing 失效。

## 5.3 East 可能是“容量充足型”实例，West 更像“结构敏感型”实例
- East：更严后仍能全部清空，只是更依赖滚动延后
- West：更严后更容易产生真实 service loss

这对 thesis 写作很有价值，因为它支持按实例 regime 分类讨论算法效果，而不是期待单一结论覆盖所有实例。

---

## 6. 当前仍需谨慎解释的点
注意：strict config 当前主要是**收紧行为与惩罚**，但还不是彻底 hard-feasible depot execution model。
所以当前 strict results 应表述为：

- `stricter-execution-oriented baseline`
- 而不是 `final fully business-feasible result`

也就是说，这一轮结果已经比 optimistic baseline 更可信，但还没到论文最终封版口径。

---

## 7. 论文层面的建议表述

### 可以写
- optimistic baseline 与 stricter realism baseline 之间存在可观差异；
- 差异在 West / Herlev 上更明显，在 East 上更小；
- 这说明 depot-aware realism 对最终 service claim 有实质影响，必须显式纳入实验解释。

### 不建议写
- “当前 strict 版本已经完全等价于真实业务执行模型”
- “100% / 99%+ SR 已可直接作为最终 business claim”

---

## 8. 当前结论
1. `stricter realism config v1` 已经成功形成三实例对照组；
2. 它证明 optimistic baseline 确实偏乐观，但没有推翻主架构；
3. Herlev / West 对 stricter realism 更敏感，East 更稳；
4. 这组 optimistic vs strict 对照，已经很适合作为 thesis 里“结果可信度与执行现实性”章节的核心素材。
