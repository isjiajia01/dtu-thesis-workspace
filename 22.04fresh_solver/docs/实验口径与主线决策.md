# 实验口径与主线决策

## 1. 主线算法架构
当前 thesis 主线固定为：

**Controller-led rolling planning + daily constructive routing backend + depot-aware repair**

即：
1. 上层 `controller` 控制 admission / protection / clipping
2. 中层 `routing backend` 解决单日多趟富约束路径
3. 下层 `depot-aware repair` 修复 gate / picking / staging / trip2 冲突

## 2. 明确不作为当前主线的方向
以下方向可作为讨论或 future work，但不作为 thesis 主实验主线：
- giant MIP / monolithic exact optimization
- pure RL / end-to-end neural combinatorial optimization
- 完整 3D packing / extreme-point 装箱主线
- 以聚类并行为主贡献点的工程化框架

## 3. 推荐实验分组
### Group A: Routing baseline
- no controller / weak controller
- routing only
- routing + minimal repair

### Group B: Controller ablations
- deadline protection
- request-age promotion
- protected reservation
- flex clipping
- plan stability penalty
- service-first vs throughput-first admission

### Group C: Depot-repair ablations
- no repair
- departure smoothing only
- overload-focused reassignment
- split / rollback / refill variants

## 4. 核心评价指标
### 服务类
- assigned orders
- on-time service rate
- overdue / expired count
- deferred count

### routing 类
- route count
- travel time
- travel distance
- trip2 usage
- unassigned reason breakdown

### depot 类
- depot penalty
- overload bucket count
- worst bucket severity
- gate overload count
- picking overload count
- staging peak violation

### 系统类
- runtime
- plan stability / churn

## 5. 建议论文叙事
- Herlev：更像 routing + repair 可取得高质量结果的较稳 regime
- East / West：更适合展示 controller 与 admission discipline 的决定性作用
- 如果某个实验显著降低 depot penalty 但损失部分 throughput，应明确写成 trade-off，而不是简单判定失败
